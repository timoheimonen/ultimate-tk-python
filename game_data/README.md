# Game Data Packaging

The Python port must run using only data inside this repository layout.

## Rule
- After migration is complete, the original repository root asset structure is not required for the Python version.
- Runtime asset loading must use `game_data/` paths only.

## Planned Source -> Destination Mapping
- `EFPS/` -> `game_data/efps/`
- `FNTS/` -> `game_data/fnts/`
- `LEVS/` -> `game_data/levs/`
- `MUSIC/` -> `game_data/music/`
- `WAVS/` -> `game_data/wavs/`
- `PALETTE.TAB` -> `game_data/palette.tab`
- `OPTIONS.CFG` -> `game_data/options.cfg` (or generated default on first run)

## Acceptance Check (End of Conversion)
- Python game starts and plays without reading assets from repository root.
- Temporarily renaming root asset folders does not break Python runtime.
- Asset inventory and gap status are tracked in:
  - `game_data/asset_manifest.json`
  - `game_data/asset_manifest_gap_list.md`
  - (generated by `tools/asset_manifest_report.py`)
